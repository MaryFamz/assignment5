BASIC INFORMATIONS ABOUT HTML

SEMANTIC HTML

The HTML semantics refers to the tags that provide meaning to an HTML page, rather than just presentation.
It makes HTML more comprehensible by better defining the different sections and layout of web pages.

INTERNAL STYLING

Internal styling is actually done by placing the stle element in the Head element but some have actually tried placing it in the body element and it worked.
WHY?

  • This is because it's also part of the HTML component/part.
  • The main point of putting the CSS in the heade is to eliminate the latency in the network, i.e there is no network request for the CSS- so if the CSSOM building starts immediately.
  • Also, adhering to web standards by using a style sheet makes for a better website and when you need help because things on your page arn't exactly that way you want them, placing your CSS in the head as opposed to the body makes for much better troubleshooting by yourself and for anyone you ask for help from.
  • Putting the style in body element also works but when shown in the browser, it lets the CSS declare before the body starts and it means your style has loaded already.
  • It loads very quickly that the users see something appear on their screen(e.g background colors).
  • If not, users see blank screen for some time before the CSS reaches the user.
